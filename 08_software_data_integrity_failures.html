Secure Third-Party Script Loading

This example demonstrates how to reduce software and data integrity risk when loading a third-party JavaScript file.

The original vulnerable example loaded a remote script with no integrity check. This fixed version keeps the secure script tag as an example and also shows visible content on the page so the file is useful when opened locally.

Protected Script Tag

<script
  src="./08_local_demo.js"></script>

The script tag above now points to a local file in this same folder, so the example can be opened and tested entirely on your computer.

Why This Is More Secure

Subresource Integrity helps the browser verify that the downloaded file matches the expected cryptographic hash. The Content Security Policy in the page also restricts where scripts can load from.

Important Note

This local version is for safe testing in your assignment folder. In a real production site, you would use a trusted external source together with a real SRI hash for that exact file version.
